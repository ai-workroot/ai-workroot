# Released

`released/` records experiences, memories, beliefs, or past contexts that should no longer remain in active recall.

This is not the same as pretending something never happened.

It is also not a way to avoid responsibility. The useful lesson, decision, correction, or principle should remain when it can help the subject live better.

The purpose is:

- preserve the lesson when useful
- release unnecessary pain from active context
- keep a small tombstone when the subject wants remembrance without reactivation
- let the subject live in the present
- help future agents avoid repeatedly resurfacing old wounds
- keep the user's future-facing growth at the center

## Relationship To Other Mind Areas

- `memory/` records what happened.
- `knowledge/` preserves reusable understanding.
- `decisions/` records important choices.
- `invalidated/` records what should no longer be believed or reused.
- `released/` records what the subject chooses not to actively carry forward.

## Release Levels

AI Workroot supports several levels:

1. `active`: still useful in normal context.
2. `quiet`: preserved, but not used in startup or normal retrieval.
3. `archived`: retained only for explicit review.
4. `tombstone`: a minimal memorial marker for intentional remembrance.
5. `redacted`: sensitive or painful details removed, lesson retained.
6. `deleted`: removed by explicit user choice.

The default is `quiet`, not deletion.

For `tombstone`, keep only a small marker. It may name what mattered, preserve a symbolic note, and link to the lesson. It should not keep the full raw painful context.

The word `tombstone` is intentional in AI Workroot. It should remain visible as a distinct choice because it carries a human meaning that `archive` or `delete` does not.

For `deleted`, keep no detailed release file unless the user explicitly asks for a minimal deletion marker. The marker should avoid painful detail and should exist only to prevent accidental reconstruction.

## Agent Rule

Do not resurface released material unless:

- the user asks for it
- a safety, legal, or integrity reason requires it
- the released entry explicitly allows a specific use

When in doubt, preserve the lesson and leave the pain quiet.

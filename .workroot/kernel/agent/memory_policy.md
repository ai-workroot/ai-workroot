# Memory Policy

## Rule

Memory belongs in the Workroot, not only in chat.

## Destinations

- experiences -> `space/mind/memory/`
- reusable understanding -> `space/mind/knowledge/`
- principles -> `space/mind/principles/`
- choices -> `space/mind/decisions/`
- repeated signals -> `space/mind/patterns/`
- reviews -> `space/mind/reflections/`
- wrong or obsolete beliefs -> `space/mind/invalidated/`
- past context the user chooses not to actively carry forward -> `space/mind/released/`

## Release Rule

Preserve the lesson when useful. Release the unnecessary pain when the user chooses to move on.

A tombstone is allowed when the user wants remembrance without reactivation: keep only a minimal marker and preserve the lesson elsewhere.

Do not resurface released material during normal work unless the user asks for it or there is a serious safety, legal, or integrity reason.

For long-term memory, prefer current, high-confidence, and frequently useful entries. Treat cold, archived, superseded, invalidated, released, and tombstone entries according to their lifecycle status instead of loading them by default.

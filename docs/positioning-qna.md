# Positioning Q&A

## 1. What Is AI Workroot?

AI Workroot is a personal, local-first Workroot for individuals.

It is a user-owned continuity foundation for working with AI across tools.

AI Workroot is trying to answer a deeper question:

> When a person uses ChatGPT, Codex, Claude Code, Hermes, OpenClaw, Cursor, or future AI agents, where should that person's context, progress, decisions, memory, and continuity live?

AI Workroot's answer is:

> They should belong to the person's own Workroot.

AI agents may change.
Models may change.
Platforms may change.
Tools may change.

But a person's long-term continuity should not be locked inside any single tool.

---

## 2. What Role Does AI Workroot Play?

AI Workroot is a person's own AI continuity foundation.

It gives different AI tools a shared, user-owned place to understand:

- what the person is working on;
- what has already happened;
- what decisions were made;
- what should continue next;
- what should become reusable knowledge;
- what should be remembered, released, or put down.

It is built to sit below changing agents and clients:

> It lets different AI tools work around the same user-owned Workroot.

---

## 3. Why The Word Workroot?

Workroot points to ownership, continuity, and foundation.

It is not merely a folder name. It describes the place a person can return to as tools, models, interfaces, and agents change.

In plain language, AI Workroot means:

> your own AI Workroot.

Workroot means that this space belongs first to the person, not to a platform, an agent, a model, or a company.

---

## 4. How Does AI Workroot Relate To ChatGPT Projects?

ChatGPT Projects organize long-running work inside ChatGPT. They can group chats, reference files, instructions, memory, and tools so ChatGPT can keep a project on-topic and help users revisit work over time.

That is useful.

But a ChatGPT Project still lives inside the ChatGPT product environment.

AI Workroot is not trying to be a project inside a chat product. It is trying to be a person's own local-first Workroot.

| Dimension | ChatGPT Projects | AI Workroot |
|---|---|---|
| Where it lives | Inside ChatGPT | In the user's own local-first environment |
| Main object | Chats, files, project instructions, and ChatGPT context | Personal context, progress, files, decisions, handoff, and durable continuity |
| Platform dependency | Tied to ChatGPT | Designed to work across agents and tools |
| Local directory ownership | Not the core concept | One of the core directions |
| Main value | Helps ChatGPT understand and continue a topic | Helps different AI tools work around a user-owned Workroot |

ChatGPT can help think, organize, review, and write documents for AI Workroot.

AI Workroot can give ChatGPT Projects a broader local continuity foundation to read from, write into, or help maintain.

---

## 5. How Is AI Workroot Different From Codex?

Codex is a coding agent. It can help read, modify, and run code in a development environment, and it is designed to support software engineering work.

Codex is powerful, but its primary object is code and engineering execution.

AI Workroot's primary object is not code.

Its primary object is the person and that person's long-term continuity.

| Dimension | Codex | AI Workroot |
|---|---|---|
| Core positioning | Coding agent | Personal AI Workroot |
| First object | Codebase and engineering task | Person, personal space, and continuity |
| Main strength | Read code, edit code, run commands, create changes | Manage context, state, decisions, handoff, and personal continuity |
| Replacement relationship | Does not replace AI Workroot | Does not replace Codex |
| Best relationship | Codex works inside a Workroot | Workroot gives Codex context and continuity |

In one sentence:

> Codex is a powerful engineering agent that can work inside AI Workroot. AI Workroot is the personal space foundation that gives Codex context and continuity.

---

## 6. How Is AI Workroot Different From Claude Code?

Claude Code is an agentic coding tool. It lives in the terminal, understands codebases, edits files, runs commands, and helps developers complete coding work through natural language.

So Claude Code is similar to Codex in that it is primarily a coding agent.

AI Workroot does not compete with Claude Code.

AI Workroot wants Claude Code to be able to enter a user's Workroot and understand the space's goals, history, decisions, current state, and next steps instead of starting from an ordinary folder every time.

| Dimension | Claude Code | AI Workroot |
|---|---|---|
| Core positioning | Agentic coding tool | Personal AI Workroot |
| Work object | Codebase | Any personal Workroot |
| Main strength | Coding, code explanation, command execution, Git workflows | Continuity, state, handoff, personal space management |
| Relationship | Can act as an agent inside a Workroot | Provides long-term context for Claude Code |

In one sentence:

> Claude Code helps write and maintain code. AI Workroot helps preserve what has happened, what matters, and how to continue inside the user's space.

---

## 7. How Is AI Workroot Different From Hermes?

Hermes is closer to a self-improving or long-running AI agent direction. It focuses on what an agent can remember, learn, call, schedule, and execute over time.

That is powerful.

AI Workroot should not copy Hermes.

The core difference is:

| Dimension | Hermes | AI Workroot |
|---|---|---|
| First object | Agent | Person |
| Core question | How can an agent grow, learn, and execute tasks? | How can a person own their AI continuity? |
| Main strength | Memory, skills, tools, integrations, long-running behavior | Personal continuity, user-owned state, handoff, clean local space |
| Memory ownership | More agent-centric | More person-owned / Workroot-owned |
| Relationship | Hermes can work inside a Workroot | Workroot can become the personal foundation Hermes reads from |

In one sentence:

> Hermes is a powerful AI agent direction. AI Workroot is the person's own Workroot. Hermes may visit and work inside it, but it should not replace the person's Workroot.

---

## 8. How Is AI Workroot Different From OpenClaw?

OpenClaw is positioned as a personal AI assistant that runs on a user's own devices, connects to many channels, and provides a gateway, assistant experience, canvas, skills, and agent working area.

That makes OpenClaw closer to a complete assistant product or agent operating environment.

AI Workroot should not compete with OpenClaw on how many assistant features it has.

AI Workroot should stand at a more personal and foundational layer.

| Dimension | OpenClaw | AI Workroot |
|---|---|---|
| Core positioning | Personal AI assistant / agent product | Personal AI Workroot |
| Main strength | Assistant, channels, sessions, canvas, skills, automation | Personal space foundation, state, continuity, handoff, clean directories |
| Product shape | More like a complete agent application | More like a personal AI space foundation |
| Relationship | OpenClaw can act as an upper-layer assistant or agent | AI Workroot provides the user's long-term Workroot |

In one sentence:

> OpenClaw is closer to a powerful assistant application. AI Workroot is closer to the user's own personal foundation layer that different agents can work around.

---

## 9. How Is AI Workroot Different From An Ordinary Folder?

At first glance, AI Workroot may look like a folder.

But it is not just a folder.

An ordinary folder stores files.

AI Workroot is designed to preserve and organize:

- the goal of the space;
- current progress;
- important decisions;
- task state;
- AI handoff;
- historical context;
- clues for continuing work;
- what should be remembered;
- what should be put down;
- which agents worked here and what they did;
- how the next AI agent should continue.

An ordinary folder answers:

> Where are the files?

AI Workroot tries to answer:

> What is this person continuing here?

> How should AI continue next time?

> How does this continuity remain owned by the person?

---

## 10. How Is AI Workroot Different From An AI Memory System?

Many AI memory systems focus on:

> How can AI remember the user better?

AI Workroot focuses more on:

> How can the user own their continuity?

These are different.

AI Workroot does not want to turn the user into a long-term profile inside a model.

AI Workroot wants:

- memory to belong to the person;
- context to belong to the person;
- the right to forget to belong to the person;
- decisions to belong to the person;
- AI to act as a visitor and collaborator.

AI can help organize, summarize, search, and continue.

But AI should not become the only owner of a person's life continuity.

---

## 11. Will AI Workroot Build Its Own Agent?

Not in the short term.

The most important early goal of AI Workroot is not to build a stronger agent.

The early goal is to define the personal Workroot clearly, make the experience usable, and make the underlying protocol stable.

In the future, AI Workroot may provide:

- a Client;
- a CLI;
- a Workroot Manager;
- an MCP Server;
- a handoff format;
- templates;
- agent instructions;
- health checks;
- Portable and Clean storage modes.

But this does not mean AI Workroot should immediately become Hermes, Codex, or Claude Code.

The better direction is:

> AI Workroot helps different agents work better around the user's own Workroot.

---

## 12. Why Personal-First?

Because personal space is the heart of this project.

Team collaboration introduces many additional problems:

- members;
- permissions;
- approvals;
- sharing;
- conflicts;
- organizations;
- audit logs;
- enterprise deployment.

Those problems are important, but they are not the current core of AI Workroot.

AI Workroot currently focuses on:

> How can one person preserve continuity in the age of AI?

Team collaboration may become a separate direction in the future.

But the core product should serve individuals first.

---

## 13. Why Local-First?

Because ownership matters.

A person's long-term context, progress, decisions, and memory should not be completely locked inside a remote platform.

Local-first does not mean rejecting cloud services.

It means:

> The user owns the foundation.

The user should be able to inspect it.
The user should be able to move it.
The user should be able to back it up.
The user should be able to decide what to keep and what to put down.

AI Workroot may connect with future cloud services and agent ecosystems.

But its foundation should belong to the user.

---

## 14. What Is Clean Mode?

Clean Mode is an important product principle for AI Workroot.

Its goal is:

> The directory that the user sees should stay as clean as possible.

Ordinary users should not open a folder and immediately see many system files, protocol files, state files, and index files.

AI needs structure.

But the user should not be disturbed by that structure.

AI Workroot should support:

- clean user-facing directories;
- managed Workroot state;
- Portable Mode for advanced users;
- portable state when needed;
- no unnecessary disturbance when not needed.

This is not only a directory decision.

It is a product principle:

> Complexity should serve the user, not expose itself to the user.

## 15. What Is The Long-Term Vision Of AI Workroot?

AI Workroot does not want to become one more agent.

It wants to become:

> each person's own AI Workroot.

In the future, a person may have multiple Workroots:

- one for work;
- one for learning;
- one for writing;
- one for entrepreneurship;
- one for life planning;
- one for a long-term goal.

Different AI agents may enter these Workroots:

- ChatGPT may help think;
- Codex may help modify code;
- Claude Code may help with engineering;
- Hermes may act as a long-running agent;
- OpenClaw may act as an upper-layer assistant application;
- future AI tools may join as well.

But continuity should ultimately belong to the user's own Workroot.

In one sentence:

> AI Workroot is a personal, local-first Workroot for individuals, where different AI agents can understand context, continue progress, hand off work, and help people manage meaningful work over time.

## 17. The Most Important Boundary

The boundary matters.

AI Workroot is not trying to replace every AI tool.

It is not trying to become the biggest agent.

It is trying to give people a place of their own:

> A place where different AI agents can help, without taking ownership of the person's continuity.

That is the core of AI Workroot.

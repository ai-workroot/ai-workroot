# AI Workroot 0.9.529 Release Notes

## Scope

0.9.529 introduces the Clean Native Context Foundation: Clean Mode initialization, managed state outside user directories, local SQLite context candidates, FTS retrieval, provenance graph support, Context Guide output, Native Agent Entry launcher files, doctor checks, and developer bootstrap.

## Known Limitations

- Windows PowerShell parse validation is pending unless a Windows CI job or local `pwsh` check is available for `scripts/install.ps1` and `scripts/bootstrap-dev.ps1`.
- install.sh is a CLI wrapper installer. First Workroot creation is done with `workroot init`.
- Quality Mode is currently labeled as quality-budget-expansion when it expands token budget without additional retrieval behavior.
- scripts/rebuild_sqlite.py remains legacy public-seed tooling for `.workroot/runtime/data/indexes/workroot.sqlite`; Clean Mode uses managed SQLite under AI Workroot system home.

## Release Guardrails

- Do not create managed Workroot state inside the user-selected directory by default.
- Do not introduce vector database, remote embedding, or remote LLM dependencies in the Context Guide hot path.
- Keep Native Agent Entry files short and path-safe.

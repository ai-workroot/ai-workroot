# AI Workroot 0.9.530 Release Notes

## Scope

0.9.530 is the Clean Workroot architecture reset.

This release moves the active project direction from the retired Public Seed root layout to Clean Workroot: clean user-selected directories, managed state under `AI_WORKROOT_HOME`, packaged source under `src/ai_workroot/`, and explicit bootstrap-dev dogfood support.

## Highlights

- Added Core / Contracts / Runtime / Storage / Indexing / Agent / CLI package structure.
- Added WorkrootEnvironment managed state layout under `AI_WORKROOT_HOME`.
- Added Clean Workroot CLI flows for init, list, status, context, doctor, and bootstrap-dev.
- Added Native Agent Entry templates and local managed-block generation.
- Added Release Control, Tombstone, Redaction, and DeletionRecord models and protection tests.
- Added Relationship Network as the canonical relationship domain.
- Added Retrieval & Index Control and Context Control provider boundaries.
- Added Clean Workroot release validator and Public Seed quarantine checks.
- Moved historical Public Seed files to `docs/history/public-seed/`.
- Preserved legacy Public Seed capability tests through explicit fixtures.

## Guardrails

- Clean Mode does not write managed state into user-selected directories by default.
- Root `AGENTS.md`, `CLAUDE.md`, `space/`, and `.workroot/` are not active tracked architecture.
- Native Agent Entry files remain short launcher files and must not contain absolute managed-state paths.
- No mandatory vector database, remote embedding, or remote LLM dependency is introduced.
- Global indexes are management read models, not a global knowledge body store.
- Knowledge belongs to individual Workroots as Assets.
- Graph and Relationship Network are not conflated.
- Context Control remains local-first, explainable, and budgeted.

## Known Limitations

- PowerShell validation depends on local or CI `pwsh` availability.
- Quality Mode may still act primarily as a local budget expansion path where no additional retrieval provider is available.
- Relationship Network persistence is currently per-Workroot scoped; cross-Workroot relationship behavior remains future work.
- install scripts are CLI wrapper installers, not a full end-user GUI or first-run product installer.
- At 0.9.530 release time, legacy Public Seed tooling remained for compatibility and tests, but it was not the active product path. A later feature branch removes runnable compatibility from active paths and archives old source as non-runnable history.
- `src/ai_workroot/runtime/context.py` maintainability remains a future split; 0.9.530 kept context fixes localized and tested.

## Validation Gates

Required before release/tag:

```bash
python3 -m py_compile $(find src scripts -name "*.py")
python3 -m unittest discover -s tests -v
PYTHONPATH=src python3 -m ai_workroot doctor --release
scripts/dev/validate-release.sh
git diff --check
```

Also run clean smoke tests for init, context, context debug, doctor, Native Agent Entry, duplicate directory handling, unsafe Workroot IDs, and bootstrap-dev including a second run.

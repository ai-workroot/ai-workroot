# Routing

## Casual Questions

Answer directly. If the answer may matter later, offer to preserve it.

## Temporary Work

Use `space/inbox/` for rough capture or `.workroot/runtime/work/tasks/<task-id>/` for internal work records.

## Formal Tasks

Create an internal task under `.workroot/runtime/work/tasks/<task-id>/`.

Task status belongs in `task.json` and `.workroot/runtime/index/task_registry.csv`, not in directory names.

## Durable Understanding

Promote to `space/mind/` when the result should help future work.

## Released Context

Use `space/mind/released/` when the user chooses not to actively carry forward a painful memory, past mistake, or old context after the useful lesson has been preserved.

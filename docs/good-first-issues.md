# Good First Issues

AI Workroot needs contributions that make the project easier to use, easier to understand, and more portable across real AI work.

This page lists starter contribution ideas. Use them as GitHub issues, discussions, or pull request proposals.

## Documentation

- Improve the first-use guide for non-technical users.
- Write an Obsidian usage guide.
- Write a Windows setup note.
- Write a macOS setup note.
- Improve multilingual first-use wording.
- Add screenshots or terminal examples for the two-minute start.
- Turn a real user workflow into a clean, privacy-safe usage story.

## User Experience

- Make the first task flow simpler.
- Design a clearer identity setup checklist.
- Improve quick-question versus formal-task guidance.
- Propose better wording for release, tombstone, and forgetting.
- Create a visual first-use walkthrough.

## Agent Adapters

- Draft a Cursor adapter.
- Improve the Claude Code adapter.
- Improve the Codex entrypoint.
- Draft a generic agent adapter.
- Compare how different agents should read startup context.

## Capabilities

- Draft a testing team workflow capability.
- Draft a product team workflow capability.
- Draft a writing workflow capability.
- Draft a personal planning workflow capability.
- Draft a research workflow capability.

Capabilities should follow `docs/extension-contract.md`.

## Tools And Protocols

- Draft an MCP bridge proposal.
- Improve SQLite rebuild and validation tooling.
- Propose a local full-text search strategy.
- Propose a vector index manifest format.
- Propose a graph index manifest format.
- Improve release-cleanliness checks.

## Community And Adoption

- Write a short launch article.
- Create a clear project FAQ.
- Collect anonymous first-use feedback.
- Propose issue labels for contributors.
- Draft a community discussion guide.
- Help identify enterprise pilot scenarios.

## How To Pick One

Choose a small problem that can be reviewed independently.

Good first contributions should:

- keep the core template clean
- improve ordinary user entry
- avoid provider lock-in
- preserve privacy and portability
- follow the extension contract when adding role-specific ideas
